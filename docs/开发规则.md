# Golden Project 开发规则

## 1. 总原则

Golden Project 面向企业线下人员管理，稳定性优先于速度。

因此所有开发遵守：

- 先守住主数据，再优化体验
- 先守住边界，再增加能力
- 先做可回滚改动，再做大范围重构
- 先补规则，再补功能

## 2. 代码放置规则

### controller

只负责：

- 接收请求
- 参数校验
- 调用 service / use case
- 返回响应

禁止：

- 直接写复杂业务
- 直接写 Prisma 查询
- 用 `any` 接收核心业务请求体

### service

只负责：

- 业务用例编排
- 状态流转
- 规则校验
- 调用 repository / adapter

禁止：

- 同时承担数据库、文件系统、外部调用、页面聚合四类职责
- 持续膨胀成超大 service

### repository / dao

只负责：

- 数据访问
- 复杂查询封装

禁止：

- 写业务规则
- 直接被 controller 调用

### adapter / integration

只负责：

- 第三方协议转换
- 调用内部领域服务

禁止：

- 在适配层重新定义任务、人、项目规则

### dto

必须用于：

- 所有核心写接口
- 所有会影响主数据或运行态的请求

禁止：

- `@Body() dto: any`
- 用前端临时 JSON 结构直接充当领域契约

## 3. 主数据与运行态规则

### 主数据

下列对象属于主数据：

- 项目
- 成员
- 模块
- 任务
- 任务日志
- 通知
- 风险
- 事件

规则：

- 主数据变更必须经过明确 service
- 必须有状态流转约束
- 关键变更必须可审计

### 运行态

下列对象属于运行态：

- 结构树 UI 状态
- 身份认领状态
- intake 快照
- 页面辅助缓存

规则：

- 运行态不能替代主数据
- 运行态字段变更必须记录用途和消费者
- 运行态结构不能由多个入口随意各写一套

## 4. 接口规则

### 写接口

必须满足：

- DTO 校验
- 明确权限
- 明确返回格式
- 明确副作用

### 读接口

必须区分：

- 列表
- 详情
- 聚合视图
- 运行态查询

禁止把所有读取能力都塞进一个大而全接口。

### 统一返回

建议逐步统一为：

- 成功：稳定数据结构
- 失败：稳定错误结构

至少要保证：

- 前端和小程序不依赖字符串猜错误
- 集成方能区分参数错、权限错、状态错、系统错

## 5. 配置规则

必须做到：

- 环境变量集中读取
- 配置集中定义
- 敏感值不写默认业务值
- 开发、测试、生产配置可区分
- 子进程或外部命令执行只允许传入显式白名单环境变量

禁止：

- 在业务 service 中零散读取 `process.env`
- 删除密码、密钥、token 设置危险默认值

## 6. 权限规则

高风险操作必须做权限校验：

- 删除项目
- 修改项目结构
- 变更成员角色
- 修改任务负责人
- 推送外部消息
- 触发 Agent 自动动作
- 读取或打开项目文件

原则：

- 角色权限和资源权限分开
- 谁能做什么必须能在文档里说清楚
- webhook / callback / 系统回调入口可以不挂用户态 guard，但必须保留内部校验与审计记录

## 7. 文件系统规则

项目资料目录属于外部资源，不应和核心项目 service 强耦合。

规则：

- 文件目录访问通过专门 service / adapter
- 文件路径必须经过安全校验
- 文件系统失败不能污染主数据事务
- 文件模板、项目资料、运行时快照分目录管理

## 8. 第三方集成规则

飞书、微信、Agent、小程序都属于接入渠道。

规则：

- 先落库，后外发
- 先归一化，再入领域
- 失败必须可重试、可追踪
- 任何渠道不得直接改写主数据语义

## 9. AI 能力规则

AI 只做辅助识别、建议、摘要、归纳。

禁止：

- 让 AI 直接成为主数据真相
- 让 AI 跳过人工 / 规则校验直接完成关键写操作

要求：

- AI 输入来源可追踪
- AI 输出结果可复核
- AI 失败不阻断核心业务链路

## 10. 变更规则

### 可以快速迭代的部分

- 页面样式
- 展示布局
- 文案
- 看板 / 日历 / 时间线等视图
- Agent 交互方式

### 必须走基线评审的部分

- 主数据模型变更
- 状态机变更
- 权限模型变更
- 多入口写入链路变更
- 外部集成协议变更

## 11. 测试与回归规则

每次影响核心链路的改动，至少回归：

- 项目创建
- 成员关联
- 任务创建 / 分配 / 更新
- 小程序读取任务
- 外部消息接入
- 运行态读写

目标不是一开始追求全量自动化，而是先建立稳定冒烟集。

## 12. 文档更新规则

下列改动必须同步更新文档：

- 新增主领域对象
- 修改核心状态流转
- 调整权限
- 调整部署链路
- 新增关键入口
- 新增跨端依赖

结论：

Golden Project 后续开发允许“板块持续演进”，但不允许“底层规则持续漂移”。
